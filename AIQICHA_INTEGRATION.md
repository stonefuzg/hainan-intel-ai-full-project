### 3. 错误处理

爬虫遇到以下情况时返回空列表`[]`且不虚构数据：

- **Timeout错误**: 爱企查响应超时
- **ConnectionError**: 网络连接问题
- **HTTPError**: 服务器返回错误状态码
- **ParseError**: HTML解析失败（无法找到数据）
- **其他异常**: 所有未预期的错误

**错误消息示例**:
```
[企业爬虫信息] 无法从爱企查解析到企业数据，尝试备选本地数据源...
[企业爬虫错误] Timeout: 爱企查响应超时，请检查网络连接
```

## 🔄 数据源管理系统集成

**文件: `data_source_manager.py`**

实现了4层自动fallback机制：

```
层级1 (主要)    : 网络爬虫 (爱企查)
    ↓ 失败
层级2 (可靠)    : 本地JSON文件 ✓ (已成功)
    ↓ 失败
层级3 (预留)    : 数据库历史记录
    ↓ 失败
层级4 (最后)    : 空列表[] + 错误消息
```

### 执行流程示例
```
[enterprises] 尝试从网站爬虫获取数据...
[企业爬虫信息] 无法从爱企查解析到企业数据，尝试备选本地数据源...
[enterprises] 尝试从本地文件获取数据...
[enterprises] ✓ 从文件成功获取 6 条数据
```

### 透明度透明设计
每条企业记录都包含`data_source`字段，完整追踪数据来源：

```json
{
  "name": "海南正尔建筑有限公司",
  "industry": "建筑业",
  "capital": 5000,
  "registration_date": "2026-02-28",
  "region": "海口市",
  "data_source": "爱企查(aiqicha.baidu.com)"
}
```
